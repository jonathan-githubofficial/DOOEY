2. Scaffold ReactLynx into a scratch dir, NON-INTERACTIVELY. First discover the
   exact flags/template names (create-rspeedy is young; do not guess):
     npx -y create-rspeedy@latest --help
   create-rspeedy mirrors the Rspack-family create-* CLIs. Expected flags:
     -d, --dir <dir>          target directory
     -t, --template <tpl>     template name
     --tools <a,b>            extra tools (comma separated)
     --override               overwrite existing files
     --packageName <name>     package name
   The React+TypeScript template is expected to be named `react-ts` (confirm it
   appears in --help output). Then scaffold into scratch:
     npx -y create-rspeedy@latest --dir .scratch/scaffold --template react-ts --packageName dooey
   STOP-on-ambiguity: if --help shows different flag names, or no `react-ts`
   template, return status "blocked" with the exact --help output. Do NOT run the
   interactive wizard (it freezes an unattended run).
   Docs: https://lynxjs.org/guide/start/quick-start .

6. Tailwind v3 + Lynx preset (verified setup):
     npm add -D tailwindcss@3 rsbuild-plugin-tailwindcss @lynx-js/tailwind-preset
   - lynx.config.ts plugins array adds:

           config: './tailwind.config.ts',
           // KEEP tailwind-merge as a dep (PLAN 8) but stop the plugin from
           // scanning its raw Tailwind source (known class-flooding gotcha):

         })
   - Create ./tailwind.config.ts:
         import { createLynxPreset } from '@lynx-js/tailwind-preset';
         export default {
           content: ['./src/**/*.{js,ts,jsx,tsx}'],
           presets: [createLynxPreset({ lynxUIPlugins: { uiVariants: {} } })],
           theme: { extend: { /* token mapping - step 8 */ } },
         };
     (uiVariants replaces pseudo/data-attr selectors for component states; keep the
     default `ui-*` prefixes for now - PLAN 5.1, crib sheet.)

     and drives Tailwind itself; do NOT add a separate PostCSS pipeline.
   Docs: https://lynxjs.org/rspeedy/styling , @lynx-js/tailwind-preset (npm).
