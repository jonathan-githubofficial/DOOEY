# 4.1 - Today / Planner composition + quick-add + habits

LAYER: L4-tasks / TIER: heavy / RISK: high (signature page-flip motion; two
requested features - recurring habits and calendar-events-in-Today - do NOT
exist in the old app; see BROOM + STOP items).

GOAL: Rebuild the Planner space (the `/` route) on Lynx elements: the day's
one-shot tasks in a top-bound flip-pad, minimum-friction quick-add, and the
signed-out state - so a signed-in user opens the app, sees today's list, adds a
task, and it appears without reload.

## SPEC

Lynx crib (orchestration-framework.md "Lynx crib sheet") governs; official docs
are the tiebreaker. Element rules used throughout this layer:
- Elements, not HTML: `<view>` (was `div`), `<text>` (was `span`/`p`/`h*`),
  `<image>` (was `img`), `<scroll-view>` for scrollers. TEXT DOES NOT INHERIT
  CSS - every `<text>` sets `color`/`font-family`/size explicitly (keep the
  token classes `text-ink`, `font-display`, etc.). Ref: crib; lynxjs.org
  /rspeedy/styling.
- No `<button>`: use `<view bindtap={fn}>` (background-thread tap) or
  `<view main-thread:bindtap={fn}>` (press-linked motion) with a `<text>` child.
- No pseudo/`data-*`/`hover:` selectors: express states via the preset's
  `uiVariants` or conditional `className` (crib; /rspeedy/styling). `group-hover`
  reveal affordances (row meta, delete-on-hover) DROP to always-visible or
  tap-reveal; note each where it appears.
- Accessibility: replace `aria-label` with `accessibility-label` and
  `aria-hidden` with `accessibility-element={false}`.

1. `Today.tsx` - `Today()` wrapper unchanged in shape: read
   `useAuthStore((s) => s.isAuthenticated)`; render `<Planner/>` or
   `<SignedOut/>`. Convert the JSX to Lynx elements.

2. `SignedOut()` - a `Panel` with `Eyebrow`, a display `<text>` headline, and a
   body `<text>`. The inline `<Link to="/account">` becomes a `<text bindtap>`
   that calls the router `navigate({ to: "/account" })` (import `useNavigate`
   from `@tanstack/react-router`; memory router from L3). Keep copy verbatim.

3. `Planner()` - call `useTasksLive()`. State: `const [selected, setSelected] =
   useState(localDate())`; `const [direction, setDirection] = useState(1)`.
   Provide a MINIMAL day stepper in place of the deferred slider: two
   `<view bindtap>` controls (prev / next) using the L2 icon set
   `ChevronLeft`/`ChevronRight`, plus a "today" reset shown when
   `selected !== localDate()`. Handlers:
   `const step = (n) => { const d = addDays(selected, n);
   setDirection(n); setSelected(d); }` (import `addDays` from `../features/
   tasks/dates`). This keeps the direction-aware flip meaningful and testable
   until 5.1 restores the `WeekStrip`/`MonthView` shelf. Render order:
   stepper row, then `<PlannerBook page={selected} direction={direction}>
   <AgendaSheet date={selected} /></PlannerBook>`, then
   `<TaskComposer date={selected} />`.

4. `PlannerBook.tsx` - keep the exported constants (SPEC top). Structure:
   a. Root `<view className="relative">` with a `Rings()` layer (three
      gradient loops via `BINDING_ROW`/`BINDING_SLOT`, positioned absolute) and
      the two static under-pad `<view>`s (ports of lines 115-122) - all decorative
      `<view>`s with `accessibility-element={false}`.
   b. The flipping page: render ONE page keyed by `page`. On `page` change, mount
      the arriving page with an enter-animation class and keep the departing page
      mounted with an exit class until its animation ends. Because Lynx has no
      `AnimatePresence`, track `{prevPage, prevDir}` in state and remove the
      departing page on `bindanimationend`.
   c. Direction-aware CSS keyframes in `src/styles/` (Tailwind v3 preset supports
      keyframes; crib "CSS transitions/keyframes for enter/exit"): forward =
      arriving page fades up from beneath while the departing page tilts
      `rotateX(0deg -> 140deg)` and darkens (`filter: brightness`); back =
      arriving page tilts down `rotateX(140deg -> 0deg)` and settles. Approximate
      the old springs with a cubic-bezier OVERSHOOT easing ("settle, don't snap",
      CLAUDE.md Motion; PLAN 5.3) e.g. `cubic-bezier(0.34, 1.3, 0.4, 1)` for the
      back-flip settle and `cubic-bezier(0.5, 0.05, 0.75, 0.55)` for the forward
      peel. Set `perspective: 1400px` on the root and `transform-origin: top` on
      the page (inline style is fine; the values are constants).
   d. Height: DROP the ResizeObserver glide. The root sizes to the arriving
      page's content (no fixed height). Accept a small non-glide when pages
      differ in height (60fps polish is PARKED).
   e. Reduced-motion: gate the keyframe classes on `useReducedMotion()` (SPEC 8)
      - when reduced, swap to a plain opacity cross-fade (crib; CLAUDE.md
      "Everything animated must also work reduced-motion").

5. `TaskComposer.tsx` - `const [open, setOpen] = useState(false)`. Render a fixed
   stamp `<view main-thread:bindtap={press}>` (classes
   `stamp-edge stamp-btn grain fixed bottom-20 right-4 z-30 ... bg-zest`) with the
   L2 `Plus` icon. Press micro-interaction via MTS (crib; lynxjs.org
   /react/main-thread-script): a `main-thread` handler that reads
   `event.currentTarget` and sets a depressed transform, releasing on the paired
   up event - keep it to `setStyleProperty('transform', 'scale(0.9)')` on down and
   `'scale(1)'` on up (see 4.3 for the shared press helper; if 4.3's helper is not
   yet landed, inline the two-line handler). The tap ALSO calls
   `setOpen(true)` on the background thread. Compute `dayLabel` via
   `toLocalNoon(date).toLocaleDateString(...)` for `accessibility-label`. Render
   `{open && <ComposerSheet date={date} onClose={() => setOpen(false)} />}` - the
   sheet's own enter/exit transition (4.3) replaces `AnimatePresence`.

6. Realtime: DO NOT add subscription code here - `useTasksLive()` (already
   ported, api.ts) subscribes to the `tasks` collection through the L1 seam
   (`lynx.EventSource`) and invalidates every tasks query on any change, so a
   `useCreateTask` write (fired from `ComposerSheet`, 4.3) re-fetches
   `useDayTasks(selected)` and the new row appears with no reload. This unit's
   job is only to mount `useTasksLive()` in `Planner()`.

7. Quick-add contract (minimum friction, CLAUDE.md "quick-add with minimum
   friction"): the FLOW is - tap the stamp -> `ComposerSheet` opens focused on
   the title input -> type a title -> confirm -> `useCreateTask` persists ->
   sheet closes -> row appears via realtime. The sheet UI itself is 4.3; this
   unit guarantees the entry point (stamp) and that `date={selected}` is passed
   so a task typed on a non-today day files under that day (payload uses
   `due_date` from the viewed date - see ComposerSheet in 4.3).

8. Reduced-motion pref (SHARED L4 prerequisite; PLAN 5.3 resolution + this run's
   decision: gate via a Zustand pref because the Lynx reduced-motion signal is
   unverified). Shape (owned by 4.3, but if this unit lands first, add it):
   extend `src/stores/theme.ts` with `reducedMotion: boolean` (default false)
   + `setReducedMotion(v)`, and export `useReducedMotion = () =>
   useThemeStore((s) => s.reducedMotion)`. All three L4 units import THIS hook,
   not `motion/react`'s.
