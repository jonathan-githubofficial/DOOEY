# 7.1 - Boards list + miniatures

LAYER / TIER / RISK: L7-boards / heavy / HIGH (first read-only render of the full
board data model on Lynx; static <svg> already proven by L2/L3, so this unit's
risk is layout fidelity + the scaled-miniature measurement, not gestures).

GOAL: Rewrite the Boards page and its list/section/miniature components on Lynx
elements so seeded boards render as a wall of cards, each showing a fan of its
pieces and a live scaled preview, with no motion/react and no DOM APIs.

SPEC (numbered, concrete):
1. `src/pages/Boards.tsx`: rewrite with Lynx elements. `<>` fragment ->
   `<view>` wrapper or fragment as the L3 page convention dictates. Render
   `<Masthead avatar={<PageDoodle page="boards" />} title="Boards" />` (both already
   Lynx). Branch on `useAuthStore((s)=>s.isAuthenticated)`: true ->
   `<BoardsList/>`; false -> the signed-out `Panel`. Replace `<h2>`/`<p>` with
   `<text>` elements and SET color + font-family + size explicitly on EVERY `<text>`
   (crib: text does NOT inherit CSS - PLAN 5.1). The "Sign in" `<Link to="/account">`
   becomes the Lynx-routing Link the L3 shell established; keep the same target.
2. `src/features/boards/components/BoardsList.tsx`:
   - `BoardsList`: call `useBoardsLive()` and `useBoards()`. Grid of cards: a create
     tile + one `BoardCard` per board + an empty-state `<text>`. Replace `<div>` grid
     with `<view>` using the same tailwind grid classes (grid, gap, sm:grid-cols-2 -
     confirm the tailwind-preset supports the responsive prefix; if not, use a fixed
     2-col at the app's breakpoint and note it). Create tile: a pressable `<view>`/
     button calling `create.mutate("Untitled board",{ onSuccess:(r)=>navigate(...) })`
     - keep the exact create-then-navigate flow.
   - `BoardCard`: wrap in the Lynx Link to `/board/$id`. Inside a `Panel`: title
     `<text>` (truncate), "Edited <date>" `<text>` (keep
     `new Date(board.updated).toLocaleDateString("en",{month:"short",day:"numeric"})`
     - Date is a JS builtin, available on PrimJS/web), the `BoardMenu`, and the fan
     of tiles.
   - `BoardMenu`: the ... options button + delete menu. Port the click-stop guards
     (the old code uses `e.preventDefault()/stopPropagation()` so the menu never
     triggers card navigation). On Lynx use `event.stopPropagation()` in the button
     handlers and a conditional-rendered menu (no `AnimatePresence`). Delete calls
     `useDeleteBoard().mutate(board.id)`.
   - `pickFanTiles`, `FanTile`, `FanContent`: copy the logic verbatim. `FanTile`
     positions via inline style `left/bottom/rotate/zIndex/aspectRatio` - Lynx
     supports these style props; keep them. `FanContent` `photo` case uses an
     `<image src={boardPhotoUrl(...)}/>` (Lynx `<image>`, not `<img>`); `doodle` case
     renders `<DoodleGlyph/>`; note/text/group/sticker/link cases become `<view>`/
     `<text>` with the same tint classes. Keep `NOTE_TINT`.
3. `src/features/boards/components/BoardSection.tsx`: port both branches. No-board
   branch: `Panel` + `Eyebrow` + copy `<text>` + an "Add a board" pressable calling
   `onAttach`. Linked branch (`LinkedBoard`): `useBoard(id)`, render title + a Link
   wrapping `<BoardMiniature board={board}/>` and the (now always-visible-muted)
   "Open board" label per BROOM ruling.
4. `src/features/boards/components/DoodleGlyph.tsx`: port to a Lynx `<svg>`. Keep the
   exact viewBox math: `vh = 100/aspect`, `viewBox="-4 -4 108 ${vh+8}"`, element size
   `style={{ width, height: width/aspect }}`. Map each stroke to a `<path>` with
   `d={strokePath(s.points)}`, `stroke={`hsl(var(--${s.color}))`}`, `strokeWidth={2.5}`,
   `vectorEffect="non-scaling-stroke"`, `strokeLinecap/Linejoin="round"`,
   `opacity={0.9}`, `fill="none"`. Cite: FETCH https://lynxjs.org/blog/lynx-3-7.html
   for the exact <svg> attribute spelling Lynx accepts (camelCase vs kebab; e.g.
   `stroke-width`/`vector-effect` may be required in kebab-case on the <svg> element -
   confirm and use whatever the 3.7 element accepts). If `vector-effect:
   non-scaling-stroke` is unsupported by the Lynx <svg>, fall back to a fixed
   `strokeWidth` in viewBox units and record the visual delta (do NOT block: this is a
   read-only glyph, not a data concern).
5. `src/features/boards/components/BoardMiniature.tsx`: this is the only non-trivial
   piece. It renders items at their real px positions inside a container scaled to
   fit, using `scale = clientWidth / CANVAS_W`. Replacement for the removed
   ResizeObserver:
   - Import `CANVAS_W`/`CANVAS_H` from `./BoardCanvas` (7.2 exports them - if 7.2 has
     not landed yet in the new tree when this unit runs, define the constants locally
     as `CANVAS_W=1600`/`CANVAS_H=1100` and let 7.2 re-home them; note it). These are
     FIXED, so the miniature aspect box is `aspectRatio: CANVAS_W/CANVAS_H`.
   - Get the rendered container width WITHOUT getBoundingClientRect: bind the
     container's Lynx layout event (`bindlayoutchange`, which reports the element's new
     size) and set `scale = width / CANVAS_W` in state; re-run on subsequent layout
     events. Cite: FETCH the Lynx element event docs
     (https://lynxjs.org/api/elements/ or the <view> event reference) to confirm the
     exact event name and payload shape (`event.detail.width` / `.height`). If a
     one-shot measurement is preferred, use the Lynx SelectorQuery
     (`lynx.createSelectorQuery().select(...).invoke({method:'boundingClientRect'})`)
     on mount instead - either is acceptable; the result feeds the same `scale`.
   - Inner scaled layer: a `<view>` sized `CANVAS_W x CANVAS_H`, `transformOrigin`
     top-left, `transform: scale(${scale})`. Render `board.doodle` as a single `<svg>`
     (canvas-px strokes, viewBox `0 0 CANVAS_W CANVAS_H`, strokeWidth 3) then each
     non-parented item via `MiniItem`/`MiniContent`. Copy `MiniItem`/`MiniContent`
     switch verbatim onto `<view>`/`<text>`/`<image>`/`<svg>`; keep `FolderShell`,
     `color-mix` fills, `NOTE_BG`/`ZONE_*` maps. It is `pointer-events-none` and inert.
6. Remove `lucide-react`/`motion/react`/`framer` imports from every new file; wire the
   2.4 Lynx icons. Confirm no `<div>/<span>/<p>/<img>/<button>` HTML tags remain -
   only Lynx elements (`<view>/<text>/<image>/<scroll-view>` etc.). If the boards grid
   needs to scroll, use `<scroll-view>` (crib).
