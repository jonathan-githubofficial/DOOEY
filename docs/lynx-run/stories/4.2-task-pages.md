# 4.2 - Task pages (structured sections on Lynx)

LAYER: L4-tasks / TIER: heavy / RISK: high (rich text editing + file upload on
young Lynx primitives; two capability gaps - YouTube iframe and file-picker -
resolved below, one as an explicit degradation, one as a STOP).

GOAL: Rebuild `/task/$id` on Lynx: a task opens to its own page with editable
title/description/due-date and the four structured sections - notes, checklist,
resources, attachments - so opening a task, editing notes, and toggling a
checklist item all persist.

## SPEC

Apply the shared element rules from 4.1 SPEC top (view/text/image, no `<button>`,
no `<a>`, `accessibility-label`, uiVariants for states, text sets its own
color/font/size). Then:

1. `Task.tsx` -> `src/pages/Task.tsx`: `const { id } = useParams({ from:
   "/app/task/$id" }); return <TaskDetail id={id} />;` Export `TaskPage` (match
   L3 route element).

2. `TaskDetail` loader: `useTasksLive()`; `const { data: task, isPending, error }
   = useTask(id)`; `error -> <Gone/>`; `isPending || !task -> return null`
   (local PB answers in ms - keep the "no skeleton flash" comment).

3. `TaskDetailLoaded`: root `<view>` with a CSS enter transition (fade + 14px
   rise, cubic-bezier overshoot settle) gated on `useReducedMotion()`. Children:
   a. Back control: `StampButton` (L2) with the `ArrowLeft` icon + "Planner",
      `bindtap` -> `navigate({ to: "/" })`.
   b. Header `Panel`: the `Check` (SPEC 5) toggling `done_at`
      (`patch({ done_at: done ? "" : new Date().toISOString() })`); title +
      description via `EditableText` (patches `title`/`description`); the
      due-date row via `<DueDateButton due={task.due_date} onChange={(due_date)
      => patch({ due_date })} />` (4.3).
   c. `<PageSections notes checklist resources onPatch={patch} attach={{...}} />`
      with `attach` wired to `useUpdateAttachments` + `attachmentUrl` (see
      STOP-3 for the disabled ADD).
   d. Board section: stub (SPEC 8).
   e. Delete: `<view bindtap>` -> `del.mutate(task.id)` then `navigate({to:"/"})`.
   `patch` = `(p) => update.mutate({ id: task.id, patch: p })` (optimistic cache
   is already in `useUpdateTask`, api.ts - do not re-add).

4. `PageSections`: port the `opened` Set + `visible`/`missing` computation
   unchanged; add-affordance dots as `<view bindtap>` with L2 icons; keep the
   "content-bearing sections always show; opened ones show this visit" behavior.

5. `Check` completion micro-interaction (shared with 4.3): press-depress on
   down/up and a tick that springs on when `done` flips true. Author with a
   `'main thread'` handler bound via `main-thread:bindtap`
   (`event.currentTarget.setStyleProperty('transform', 'scale(0.78)')` on press,
   restore on release) - lynxjs.org /react/main-thread-script; the tick `<view>`
   animates `scale(0 -> 1)` via a CSS keyframe with a cubic-bezier overshoot
   (`cubic-bezier(0.34, 1.56, 0.5, 1)`) to keep "settle, don't snap" (CLAUDE.md
   Motion). Reduced-motion -> no scale keyframe, instant show. If 4.3's shared
   press helper is landed, import it; otherwise inline the two-line MTS handler
   (identical shape) and 4.3 reconciles.

6. Input focus: Notes/Checklist/Resources take an `autoFocus` prop. Lynx has no
   `autoFocus` attribute; on mount when the section was just opened, focus the
   input imperatively via `lynx.createSelectorQuery().select("#<id>")
   .invoke({ method: "focus" }).exec()` (give the input an `id`). Guard so it
   only fires when the section was opened this visit (mirror the old
   `opened.has(key) && empty` condition).

7. `openExternal.ts` (new seam; Lynx has no `<a>`): export
   `openExternal(url: string): void`. WEB target implementation opens the URL in
   the host browser context. VERIFY the Lynx-web host's URL-open API during
   implementation (candidates: `lynx.openSchema(url)` for native schemes; a
   web-host navigation for http/https). If neither is available on the web
   target, this is a small PARKED gap (links render but do not open) - note it,
   do not block. Used by Resources (link + YouTube thumbnail) and Attachments
   (open/download).

8. Board section stub (until L7): render nothing for the board area, OR a
   disabled placeholder `Panel` is NOT allowed (no "coming soon", CLAUDE.md
   code-quality). Simplest compliant port: omit the board block entirely in 4.2
   and add a BROOM note that 7.x re-inserts it. Remove the `board` field usage
   and the `@/features/boards` import so typecheck passes without L7.

9. Id helper: add/rewire a shared `newId()` (e.g. `src/lib/id.ts`, or reuse an
   existing L1 helper if one was created) returning a unique string without
   `crypto` (timestamp + counter, or a small nanoid). Checklist/Resources use it
   for new item ids. Verify `crypto.randomUUID` before assuming it exists.
