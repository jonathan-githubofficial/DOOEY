# 1.3 - Port Zustand stores + PocketBase api modules
LAYER: L1-foundation / TIER: heavy / RISK: medium

GOAL: Port the client-state stores and all PocketBase query/mutation modules into
the new tree, routing every persisted store through a storage-seam import boundary
that unit 1.4 finalizes, so state + data logic is in place before the pb client seam.

CONTEXT: Units 1.1 (toolchain, aliases, tokens) and 1.2 (pure logic + types) are
done. The new tree has src/lib/{date,format,cn,doodle}.ts, all types.ts, and the
pure tasks/learning/style logic. There is NO pb client, storage adapter, or
useCollectionLive hook yet. This unit creates MINIMAL versions of those three seam
files so the stores/api typecheck and build green; unit 1.4 REWRITES them with the
real host resolution, EventSource wiring, and storage adapter. This ordering is a
deliberate, documented resolution of the fact that the layer runs 1.3 before 1.4 yet
1.3's modules import the pb/live/storage seam (see SPEC 1).

SPEC (numbered):

   with native host work (out of this run's mechanical reach).

   mutation semantics exactly (comment in tasks/api.ts explains the auto-cancel
   reasoning - keep it).

   other by direct module path (they already do: e.g. learning/store imports ./api).
5. Confirm no store/api file imports a view component or a not-yet-ported module.
