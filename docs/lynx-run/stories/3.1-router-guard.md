SPEC (concrete, no judgment calls):

1) Memory history (crib "Routing"; PLAN 5.2; doc
   https://lynxjs.org/react/routing/tanstack-router - FETCH it, it is the tiebreaker).
   Lynx has no History API, so the browser history plugin cannot be used. Build the
   router with an explicit memory history:
       import { createRouter, createMemoryHistory } from "@tanstack/react-router";
       const memoryHistory = createMemoryHistory({ initialEntries: ["/"] });
       export const router = createRouter({ routeTree, history: memoryHistory,
         defaultPreload: "intent", scrollRestoration: true });
   If the fetched doc gives a different import path for `createMemoryHistory` (e.g.
   from `@tanstack/history`) or a different wiring, FOLLOW THE DOC. If the doc is
   unreachable AND `createMemoryHistory` is not exported from `@tanstack/react-router`
   at the pinned version, STOP (status "blocked", exact question) - do not guess.

2) `startTransition` on ReactLynx (crib "Framework": ReactLynx is React-17-style;
   `@lynx-js/react/compat` supplies React-18 APIs). TanStack Router calls React's
   `startTransition` internally. Follow the routing doc's exact recipe for providing
   it (typically: alias/shim so the concurrent API resolves to
   `@lynx-js/react/compat`, or pass it via a router option). Implement exactly what
   the doc prescribes. If the doc does not mention `startTransition` at all and the
   build/typecheck errors on a missing `startTransition`, STOP with the error text.

3) `URLSearchParams` polyfill (crib "Routing"; PLAN 4b/8). TanStack Router's search
   parsing needs `URLSearchParams`, absent in PrimJS. Add a single side-effect import
   at the app entry (before the router is created), e.g. `import
   "url-search-params-polyfill";`. Confirm it also runs on the web target build.

4) Root route - PORT `src-legacy/router.tsx` lines 21-28 verbatim in shape:
       const rootRoute = createRootRoute({
         component: () => (<><Backdrop /><Outlet /></>),

   `Backdrop` imports from `@/features/style` (built in 2.2). Replace the RN/HTML
   fragment host only if the Lynx compiler requires it (fragments are fine in
   ReactLynx). Do NOT add markup here beyond `<Backdrop/>` + `<Outlet/>`.

7) The guard - PORT `src-legacy/router.tsx` lines 30-64 EXACTLY (this is the HIGH-RISK
   surface; port verbatim, do not reinterpret):
   - loginRoute.validateSearch: `redirect` is `string | undefined`.
   - loginRoute.beforeLoad: if `pb.authStore.isValid` -> `throw redirect({ to: "/" })`.
   - appRoute.beforeLoad: if `!pb.authStore.isValid` -> `throw redirect({ to: "/login",
     search: { redirect: location.href } })`. `location` is the TanStack `beforeLoad`
     arg (route location), NOT `window.location`. On memory history `location.href`
     is `pathname + search` (e.g. "/", "/calendar") - this is exactly the value 3.2's
     Login pushes back to. Do not substitute `window.location` (absent on Lynx).
   - appRoute.component: the authed shell = a root `<view>` with safe-area padding
     (crib: use Lynx safe-area insets, NOT `env(safe-area-inset-*)`; the concrete
     inset mechanism is defined by 2.3's safe-area work - reuse it) wrapping
     `<Outlet/>`. Do NOT add `<Dock/>` here - 3.3 adds it. Map the old
     `<main class=...>` to `<view>` with the equivalent Tailwind classes (mx-auto,
     max-w, px, pb/pt) using the preset from 2.x.
   Any place the memory-history behaviour of the guard is unclear (e.g. `location.href`
   shape, redirect throw support) that this step does not pin down: STOP, do not improvise.
