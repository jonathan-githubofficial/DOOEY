     integration model).

     what is/isn't mature yet).
   - `https://lynxjs.org/next/blog/lynx-3-7` (the `<svg>` element - for the

   Baseline facts already verified 2026-07 (re-confirm at fetch time, note any
   drift):
   - Sparkling is TikTok's open-sourced application layer for Lynx (an
     Expo-for-Lynx analogy): a CLI scaffolds a native iOS+Android app from one
     Lynx bundle; navigation is native (not a web router) via
     `sparkling-navigation`, each page in its own native container;
     container-creation time passes values into the Lynx runtime attached to
     `lynx.__globalProps`; there is a typed native bridge with codegen.
   - No EAS-equivalent cloud build/submit service exists for Lynx. Xcode +
     Android Studio produce and sign the actual binaries; PLAN section 7
     recommends **fastlane in GitHub Actions** to automate this, mirroring the
     style (not the mechanism) of this repo's existing
     `.github/workflows/deploy.yml` SSH-based auto-deploy - i.e. secrets-based,
     no new cloud-credential scheme invented just for this.
   - `<svg>` landed in Lynx 3.7 (Apr 2026): ~17 common SVG tags + 40+
     attributes, parsed off the main thread, rendered as a single native view
     per platform (Android/iOS/HarmonyOS/macOS/Windows/Web uniformly per the
     announcement) - this run's doodle canvas (PLAN 5.5, L7) and static icon set
     (PLAN 5.9, L2) both depend on this; native-side fidelity was never
     verified by this run.
   - `@font-face` on Lynx ignores `font-weight`/`font-style`/`font-variant` -
     static per-weight font files are required (PLAN 5.7); Android specifically
     has known custom-font bugs - prefer `.ttf` + `local(file://...)` and test
     on a physical Android device before shipping.
   - PrimJS (Lynx's native JS engine) lacks `TextEncoder`/`TextDecoder` - the
     web target uses the browser's, which is why this entire run could only
     prove the WEB-target PocketBase SSE round-trip (Phase 0 spike, item 1);
     native SSE decoding (`lynx.EventSource` payloads) is genuinely unverified.
