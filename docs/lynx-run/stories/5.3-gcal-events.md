    end_at       date (datetime), REQUIRED  (stored UTC ISO)

ADAPTER + TIMEZONE (concrete; resolve the day/minute math now):
- The app computes "a day" and "now" in BROWSER-LOCAL time (dates.ts localDate uses
  new Date(); TimeboxSheet useNowMinutes uses getHours/getMinutes). start_at/end_at
  are UTC. `toEvent(r)` must map a UTC event onto a local day + minutes:
    const start = new Date(r.start_at); const end = new Date(r.end_at);
    const startMin = start.getHours()*60 + start.getMinutes();   // browser-local
    const durMin = Math.max(SNAP, Math.round((end - start)/60000));
    const day = `${start.getFullYear()}-${pad2(start.getMonth()+1)}-${pad2(start.getDate())}`;
  Return AgendaExternal:
    { id: r.id, sort: startMin, done: false, title: r.title || "(busy)",
      subtitle: undefined, accentClass: "bg-sky",           // foreign = blue hue
      to: "/calendar", params: {},                          // events have no page
      onToggle: () => {}, onSort: () => {},                 // inert -- read-only
      startMin: startMin > 0 ? startMin : undefined, durMin }
  NO onSchedule/onResize/onUnschedule -> the grids treat it as read-only (no drag).
  Group by `day` for externsByDay (WeekGrid) and filter by the selected date for
  AgendaSheet/TimeboxSheet. Cite architecture "foreign events ... read-only".
- ALL-DAY events (Google `start.date`; the hook stores midnight UTC) land at
  startMin 0 -> render in the Today list (sort 0, top) but NOT the timed grid
  (WeekColumn filters startMin>0). Acceptable for parity + the @l5 gate; note it.
  Do NOT special-case all-day beyond this unless a later story asks.
