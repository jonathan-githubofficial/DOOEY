3. BoardObject MTS drag (the core replacement for `motion drag`). Each object is a
   positioned `<view>` at the canvas origin, moved by `transform: translate(x px, y px)`.
   - State: `const drag = useMainThreadRef({ startX:0, startY:0, baseX:item.x,
     baseY:item.y, curX:item.x, curY:item.y, active:false })` (all JSON-serializable
     numbers/bools per MTS constraint). Also keep item width/height as captured
     constants (`w = widthOf(item)`; height: use a per-kind estimate or the box's
     rendered height only where needed - clamp uses w for x, and CANVAS_H - a small
     margin for y; the old code only constrained via the canvas box, so a coarse clamp
     matches).
   - Bind on the object `<view>`:
     `main-thread:bindtouchstart={onDown}`, `main-thread:bindtouchmove={onMove}`,
     `main-thread:bindtouchend={onUp}` (and `main-thread:bindtouchcancel={onUp}`).
     Also `main-thread:bindtap={onTap}` for folders (toggle open). Selection
     (`onSelect`) fires from `onDown` via `runOnBackground(select)()`.
   - `onDown(e){ 'main thread'; const t=e.touches[0]; drag.current.startX=t.clientX;
     drag.current.startY=t.clientY; drag.current.baseX=<item.x>; drag.current.baseY=
     <item.y>; drag.current.active=true; runOnBackground(onSelect)(); if(section)
     runOnBackground(onDragStartItem)(); }` (item.x/item.y captured as numbers).
   - `onMove(e){ 'main thread'; const d=drag.current; if(!d.active) return; const t=
     e.touches[0]; let nx=d.baseX+(t.clientX-d.startX); let ny=d.baseY+(t.clientY-
     d.startY); nx=Math.max(0, Math.min(CANVAS_W - <w>, nx)); ny=Math.max(0, Math.min(
     CANVAS_H - 40, ny)); d.curX=Math.round(nx); d.curY=Math.round(ny);
     e.currentTarget.setStyleProperty("transform","translate("+nx+"px,"+ny+"px)");
     runOnBackground(onDragMove)(nx - d.baseX, ny - d.baseY); }` - the
     `onDragMove(dx,dy)` background callback drives drop-target highlight (loose items)
     and section-carry (sections). Throttle the background call if the gate shows jank
     (e.g. skip when the delta since last report < 4px) - correctness does not depend
     on it firing every frame.
     WHY clientX (not touches[0].x): drag is a translation, so window-space deltas are
     origin-independent and need NO element measurement - and MainThread.Element has NO
     getBoundingClientRect (FETCHED + confirmed at
     https://lynxjs.org/api/lynx-api/main-thread/main-thread-element.html : methods are
     getAttribute/setAttribute/setStyleProperty/setStyleProperties/
     getComputedStyleProperty/querySelector(All)/invoke/animate - no bounding-rect).
   - `onUp(e){ 'main thread'; const d=drag.current; if(!d.active) return; d.active=
     false; runOnBackground(onDrop)(d.curX, d.curY); runOnBackground(onDragEndItem)();
     }`. `onDrop(x,y)` is a background function that calls the existing
     `onCommit({x,y})` prop -> `handleCommit(item,{x,y})` -> folder-tuck / section-carry
     / plain `updateItem` (optimistic `useUpdateBoard`). This is the "commit position
     to PB on release" requirement.
   - Keep in sync from realtime / section-carry: the old `useLayoutEffect` that resets
     x/y from `item.x/item.y` when not dragging -> a normal `useEffect`/`useLayoutEffect`
     that, when `!drag.current.active`, sets the element transform to
     `translate(item.x px,item.y px)` (via a `useMainThreadRef` to the element or by
     making translate a rendered inline style keyed on item.x/item.y). Rendered inline
     style is simplest: base `transform` = `translate(item.x px,item.y px)` in JSX;
     the MTS move overrides it live; on commit React re-renders with the new item.x/y
     so the two agree. Prefer the rendered-style baseline + MTS override; only reach
     for a ref reset if a jump frame appears (cite the 5.2 pattern).
   - zIndex: keep the exact formula (`isZone?1 : isFolder?(open||selected?35:12) :
     selected?30:10`) as an inline style on the object `<view>`.

5. Handles (rotate + resize) via MTS. Port `Handles` to two pressable handles bound
   with `main-thread:bindtouchstart/move/end`, using a `useMainThreadRef` gesture
   record.
   - RESIZE (bottom-right): pure delta - `onDown` stores `startX=t.clientX`,
     `startY=t.clientY`; `onMove` computes `w=clampW(baseW + (t.clientX-startX))`, plus
     `h=clampH(baseH + (t.clientY-startY))` for sections, plus
     `size=round(baseSize*(w/baseW)*10)/10` for text; apply live via
     `boxRef.setStyleProperty("width", w+"px")` (and fontSize/height as needed) using a
     `useMainThreadRef` to the box element (`main-thread:ref`). `onUp` ->
     `runOnBackground(commit)({w[,h][,size]})` -> `onCommit` (clears `live`, writes
     item). Keep the exact `clampW`/`clampH` bounds and the resizable/twoD rules
     (link + group do not resize; section resizes 2-D; text scales font with width).
   - ROTATE (top): the old code needs the box CENTER in window coords
     (getBoundingClientRect) to compute `atan2`. That API does NOT exist on
     MainThread.Element. Resolution (default): on `onDown`, measure the box center once
     on the BACKGROUND thread with the Lynx SelectorQuery
     (`lynx.createSelectorQuery().select("#"+boxId).invoke({method:"boundingClientRect",
     success:(r)=>...})`, FETCH https://lynxjs.org/api/lynx-api/nodes-ref/ or the
     SelectorQuery doc for the exact call), pass `cx=r.left+r.width/2`,
     `cy=r.top+r.height/2` into the gesture ref via `runOnMainThread`; `onMove` then
     does `rot=round(baseRot + (atan2(t.clientY-cy, t.clientX-cx) - baseAngle)*180/PI)`,
     applied live via `setStyleProperty("transform", ...rotate(rot deg))` on the box.
     `onUp` -> `runOnBackground(commit)({rot})`. FALLBACK (if SelectorQuery-in-gesture
     is too slow or the method is unavailable on the web target per the spike findings):
     map the handle's horizontal drag delta to degrees
     (`rot=round(baseRot + (t.clientX-startX)*0.5)`) - a common approximation; `rot`
     stays a degrees number so the DATA SHAPE IS UNCHANGED (NOT a STOP). Record which
     path you used in the unit result. Live rotate/resize preview replaces the old
     `live` React state: mutate style directly during the gesture (no re-render), commit
     the settled numbers on release.

     `onLooseDragMove(item, dx, dy)` -> `targetAt` -> `setDropTarget(id)`; the target
     folder/section re-renders with its "hot" tint / "Drop into folder" label. This is
     a normal React re-render off a throttled background call - correct and cheap.
