   Deadline: 2026-08-01

   ```

   ## Week 1
   - [ ] Day 1 - Mon 2026-07-20 - Select statements
   - [ ] Day 2 - Wed 2026-07-22 - Joins

   ## Week 2
   - [ ] (checkmark) Day 3 - Fri 2026-07-24 - GATE 1: Query practice test
   ```
   Replace `(checkmark)` with the literal U+26F3 flag glyph used elsewhere in this
   repo's schedules (copy the exact gate glyph from any src-legacy SCHEDULE.md sample
   referenced in the skill's file-formats reference, or simply also include the
   literal word `GATE` in that line's topic - parse.ts's `isGate` check accepts

   avoids any copy/paste encoding risk). Dates are ascending, all 3 sessions are
   dated, 1 gate, no duplicate labels - all clean per verify-program's checks.

   e2e/fixtures/sql-basics/TESTS.md:
   ```

   - Write a query using an inner join and get correct results.
   ```
   (Matches verify-program's `gate[-\s]?1\b` case-insensitive regex against the
   gate's number extracted from the topic "GATE 1: Query practice test".)

   e2e/fixtures/sql-basics/calendar.ics: MUST use CRLF (`\r\n`) line endings - do NOT
   create this file with a text editor or a plain multi-line Write/echo, both are
   liable to normalize to LF and fail verify-program's explicit CRLF check. Generate
   it with a small Node snippet so the line endings are exact, e.g.:
   ```
   node -e "
   const fs = require('node:fs');
   const lines = [
     'BEGIN:VCALENDAR','VERSION:2.0','PRODID:-//DOOEY//E2E Fixture//EN',
     'BEGIN:VEVENT','UID:day1@dooey-e2e','DTSTART;VALUE=DATE:20260720',
     'SUMMARY:Day 1 - Select statements','END:VEVENT',
     'BEGIN:VEVENT','UID:day2@dooey-e2e','DTSTART;VALUE=DATE:20260722',
     'SUMMARY:Day 2 - Joins','END:VEVENT',
     'BEGIN:VEVENT','UID:day3@dooey-e2e','DTSTART;VALUE=DATE:20260724',
     'SUMMARY:Day 3 - GATE 1 Query practice test','END:VEVENT',
     'END:VCALENDAR',
   ];
   fs.mkdirSync('e2e/fixtures/sql-basics', { recursive: true });
   fs.writeFileSync('e2e/fixtures/sql-basics/calendar.ics', lines.join('\r\n') + '\r\n');
   "
   ```
   This gives DTSTART dates 2026-07-20/22/24 matching SCHEDULE.md's three sessions
   exactly (verify-program's calendar/schedule agreement check), valid
   BEGIN/END:VCALENDAR markers, and CRLF endings.
   DAILY-TEMPLATE.md and LOG.md are intentionally OMITTED - verify-program only warns
   ("No DAILY-TEMPLATE.md", "No LOG.md"), never errors, on a missing optional file;
   expect those two warnings in the output and do not treat them as failures.
