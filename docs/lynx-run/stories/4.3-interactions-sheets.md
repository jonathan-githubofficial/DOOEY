## SPEC

Element rules from 4.1 SPEC top apply (view/text/image, no `<button>`/`<a>`,
`accessibility-label`, uiVariants for states, per-`<text>` color/font/size).

1. Reduced-motion pref + shared motion tokens (this unit owns them):
   - In `src/stores/theme.ts` add `reducedMotion: boolean` (default false) and
     `setReducedMotion(v: boolean)`. Export
     `export const useReducedMotion = () => useThemeStore((s) => s.reducedMotion)`.
     4.1/4.2 import THIS (not `motion/react`). (Idempotent: if a sibling already
     added it, reuse.)
   - `src/lib/motion/easing.ts`: export named easings as strings, e.g.
     `SETTLE = "cubic-bezier(0.34, 1.3, 0.4, 1)"` (overshoot settle for
     enter/back), `PEEL = "cubic-bezier(0.5, 0.05, 0.75, 0.55)"` (accelerating
     exit), `TICK = "cubic-bezier(0.34, 1.56, 0.5, 1)"` (completion pop). These
     encode CLAUDE.md "settle, don't snap" + duration discipline (interactions
     <= 200ms, transitions <= 350ms).

2. Shared press helper (`src/lib/motion/press.ts`) - MTS (lynxjs.org
   /react/main-thread-script; crib "Main-Thread Scripting"): export two
   `'main thread'` functions bound with `main-thread:bindtap` (and the
   touch-start/end pair if press-hold feedback is wanted). Minimal shape -
   FIRST LINE of each is the `'main thread'` directive:
   ```
   export function pressDown(e) { 'main thread';
     e.currentTarget.setStyleProperty('transform', 'scale(0.92)'); }
   export function pressUp(e) { 'main thread';
     e.currentTarget.setStyleProperty('transform', 'scale(1)'); }
   ```
   Constraints to obey (from the doc): first line is the directive; captured
   vars must be JSON-serializable; runs only after TTI (no first-paint use); no
   nested function defs; cannot mutate outer-scope vars (use `useMainThreadRef`
   for cross-call state). `Check` (4.2), `TaskComposer` (4.1), and the sheet
   buttons here use these. If a component needs a scale other than the default,
   pass it via a `main-thread:` data attribute the handler reads from
   `e.currentTarget` (serializable), NOT via closure.

3. Sheet open/close/settle (`ComposerSheet`): NO `AnimatePresence`. Parent
   (`TaskComposer`, 4.1) mounts the sheet on open; on close, the sheet animates
   out then calls `onClose`. Implement:
   - Backdrop `<view>` fades opacity 0 -> 1 (transition `opacity 180ms linear`);
     `bindtap` on the backdrop closes.
   - Sheet `<view>` slides `translateY(100% -> 0)` on enter with easing `SETTLE`,
     duration ~320ms; on close, toggle an "exiting" class that slides
     `0 -> 100%` with `PEEL` and remove the sheet on `bindanimationend`
     (or `bindtransitionend`). Track an `exiting` boolean in state.
   - The grab handle is a plain header bar; tapping it OR Cancel OR backdrop
     closes. (Drag-to-dismiss = 5.2.)
   - Reduced-motion: skip the translate; use an opacity-only fade (both backdrop
     and sheet), same durations.

4. Completion + press micro-interactions: `Check` (4.2) already specifies the
   tick draw-on using `TICK`; ensure the shared `press.ts` + `easing.ts` this
   unit ships match what 4.2 imports. Task completion in the UI flows through
   `Check.onToggle` (AgendaSheet `TaskRow`, TimeboxSheet block, TaskDetail
   header) -> `useUpdateTask({ done_at })`; the done row then transitions to the
   done pile (CSS enter/exit). Verify the completion is visible AND persisted.

5. `DueDateButton` pop-open (no native picker): the chip is a `<view bindtap>`
   toggling a small `open` state. When open, render a `Panel`-styled popover of
   `<view bindtap>` chips: "Today" (`toPbDate(localDate())`), "Tomorrow"
   (`toPbDate(addDays(localDate(),1))`), "+2d / +3d / +7d", and "Clear"
   (`onChange("")`). Selecting a chip calls `onChange` and closes. The chip label
   still reads `due <dueInfo.text>` with the tone classes. Add a BROOM comment:
   "full calendar-grid date picking -> unit 5.1 (reuse `MonthView`)."

6. Keep `SheetHeading` and the binding-hole row (`BINDING_ROW`/`BINDING_SLOT`/
   `RING_COUNT` from `PlannerBook`, 4.1) so `AgendaSheet` and `TimeboxSheet`
   align with the pad rings.

7. `extern` prop: `AgendaSheet`/`TimeboxSheet` keep `extern: AgendaExternal[] =
   []` in their props and the `ExtRow`/session-boxing code paths (dead until fed
   by L5/L6), OR - cleaner per CLAUDE.md no-dead-code - trim the `ExtRow`/session
   branches and RE-ADD them in L6 when learning sessions return. DECISION: keep
   the `extern` PROP + `ExtRow` render (it is a real, typed, soon-used path the
   L6 story will feed; removing then re-adding churns the file). Do NOT wire any
   feed here.
