DATA SHAPES - PRESERVE EXACTLY (a change to any of these is STOP-on-ambiguity):
- `Stroke = { color: string, points: [number,number][] }`. `color` is a token NAME
  ("ink"|"zest"|"sky"|"clay"), rendered `stroke={\`hsl(var(--${color}))\`}`.
- BOARD-LEVEL freehand (`board.doodle: Stroke[]`): points are ABSOLUTE canvas px
  (0..1600 x 0..1100), rendered in a <svg> with `viewBox="0 0 1600 1100"`, strokeWidth
  3. (Old `pointOnCanvas` produced raw canvas px; keep that.)
- PAD-DRAWN strokes (DoodleEditor / DoodleStudio / FolderStudio): points are PERCENT
  (0..100) of the pad, rendered in a <svg> `viewBox="0 0 100 100"`. Produced by the
  MTS `pointerPct` replacement.
- PACK / PLACED doodles (`PackDoodle.strokes`, `DoodleItem.strokes` + `aspect`): the
  pad strokes passed through `normalizeDoodle` (x spans 0..100, y spans 0..100/aspect,
  `aspect = boundingW/boundingH`). Do NOT change `normalizeDoodle`.
  These three coordinate spaces are DISTINCT and load-bearing - keep them separate.

SPEC (numbered, concrete). FETCH the cited docs; do NOT guess.

1. MTS capture pattern (shared shape for every pad + the board layer). Freehand must
   NOT re-render React per pointer move (crib: MTS exists precisely for this). State:
   `const cap = useMainThreadRef({ active:false, pts:[], d:"" })` (pts = array of
   `[number,number]`, JSON-serializable). A dedicated live `<path>` element gets
   `main-thread:ref={livePathRef}`. Bind the capture surface:
   `main-thread:bindtouchstart={onDown}`, `main-thread:bindtouchmove={onMove}`,
   `main-thread:bindtouchend={onUp}`, `main-thread:bindtouchcancel={onUp}`. Docs:
   FETCH https://lynxjs.org/react/main-thread-script (directive, main-thread: binds,
   `useMainThreadRef`, `runOnBackground`, serialization/no-nested-def constraints) and
   the <svg> element page https://lynxjs.org/blog/lynx-3-7.html (supported tags/attrs).
   - `onDown(e){ 'main thread'; const p = toSpace(e); cap.current.active=true;
     cap.current.pts=[p]; cap.current.d="M "+p[0]+" "+p[1];
     livePathRef.current.setAttribute("d", cap.current.d); }`
   - `onMove(e){ 'main thread'; const c=cap.current; if(!c.active) return; const p=
     toSpace(e); const last=c.pts[c.pts.length-1]; if(Math.abs(p[0]-last[0])+Math.abs(
     p[1]-last[1]) <= THRESH) return; c.pts.push(p); c.d += " L "+p[0]+" "+p[1];
     livePathRef.current.setAttribute("d", c.d); }` - THRESH mirrors the old thinning
     (0.6 for percent pads; 2 for the board px canvas). Build the `d` string by hand in
     the handler (do NOT rely on `strokePath` being callable inside an MTS function;
     the format is identical: "M x y L x y L ...").
   - `onUp(e){ 'main thread'; const c=cap.current; if(!c.active) return; c.active=false;
     if(c.pts.length>1) runOnBackground(commitStroke)(c.pts); c.pts=[]; c.d="";
     livePathRef.current.setAttribute("d", ""); }`. `commitStroke(points)` is a normal
     background function that builds `{ color: <ink>, points }` and writes it in the
     component's existing shape (board.doodle append, or editor setStrokes, etc.).
   - `<ink>` (current color token) is captured as a serializable string; if it can
     change mid-session, read it in `commitStroke` (background) from state, not in MTS.
2. `pointerPct` MTS replacement in `src/lib/doodle.ts`. It must yield PERCENT (0..100)
   of the pad from a touch. Because MainThread.Element has NO getBoundingClientRect
   (confirmed: methods are getAttribute/setAttribute/setStyleProperty/
   setStyleProperties/getComputedStyleProperty/querySelector(All)/invoke/animate -
   https://lynxjs.org/api/lynx-api/main-thread/main-thread-element.html), use the touch's
   ELEMENT-LOCAL coordinates plus the element's computed size:
   `function pctFromTouch(e){ 'main thread'; const t=e.touches[0]; const el=
   e.currentTarget; const w=parseFloat(el.getComputedStyleProperty("width"))||1; const
   h=parseFloat(el.getComputedStyleProperty("height"))||1; const clamp=(v)=>Math.min(
   100,Math.max(0,v)); return [ Math.round(clamp(t.x / w * 100)*100)/100,
   Math.round(clamp(t.y / h * 100)*100)/100 ]; }` - `t.x`/`t.y` are the touch position
   in the touched element's coordinate system (confirmed at
   https://lynxjs.org/api/lynx-api/event/touch-event.html: each touch has x, y (element
   coords), pageX/pageY (LynxView), clientX/clientY (window)). This is the `toSpace`
   used by percent pads. Keep the exact rounding (2-decimal) to match stored data.
   Export it as an MTS-usable helper (a `"main thread"` function) that the pads import.
